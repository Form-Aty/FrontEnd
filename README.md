# 폼앗이 — Frontend

대학생 설문 품앗이 PWA. **응답은 응답으로 갚는다.**

React + Vite + TypeScript + PWA. 백엔드 연동 전이라 **더미 데이터 + 목 API**로 동작하며,
`develop_system.md`의 스키마·API·정산 로직과 `design_system.md`의 토큰·타이포·컴포넌트를 그대로 구현했다.

## 실행

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # tsc + vite build (PWA SW/매니페스트 생성)
npm run preview  # 빌드 결과 미리보기
```

데모는 시드 사용자(**김설문**)로 자동 로그인된 상태로 시작한다. 마이페이지에서 로그아웃하면
온보딩(랜딩 → 학교 인증 → 환영) 흐름을 볼 수 있다(인증 코드는 아무 6자리 숫자).

화면은 첨부 시안과 동일하게 구성했다: 온보딩 3종, 홈(원형 상호성 게이지), 설문 피드(정렬 탭),
설문 상세, 응답 중(경과 타이머), 응답 완료, 내 설문(진행중/종료됨), AI 설문 설계(4단계 위저드),
마이페이지. 하단 탭은 홈·설문 피드·(+ 등록)·내 설문·마이페이지 5구성에 중앙 FAB.

## 구현된 핵심 로직 (develop_system.md)

- **§4.1 정산**: 응답 시작(`startResponse`) → 완료(`completeResponse`)에서 중복 차단 ·
  최소 체류시간 검사 · 증빙 토큰 검증 · 응답자 적립(+)/작성자 차감(−)을 한 흐름으로 처리.
  작성자 잔액이 음수가 되면 그의 설문을 자동 `PAUSED`.
- **§3 원장 원칙**: 잔액 변경은 항상 (원장 추가 + `users` 잔액 갱신)을 함께. append-only.
- **§4.2 피드**: `w1·잔여수 + w2·기여도 + w3·(1/경과시간) − w4·소요시간` 가중 정렬.
  내 설문·이미 응답한 설문 제외.
- **§4.3 비용**: `cost = ceil(est_minutes / 2)`, 최소 1.
- **§4.4 편향 감수**: 유도/이중/모호/누락보기/척도불균형 5종 탐지, JSON 스키마 출력.
- **§7 어뷰징 방어**: 이메일 1계정, 1인 1회 응답, 체류시간, 신고 임계치 시 동결.

## 자체 설문 빌더 (구글폼 레퍼런스) — 유일한 설문 방식

외부 링크 연동은 제거했다. 모든 설문은 **자체 빌더로 만들고 앱 안에서 응답**한다(완료 자동 확인).
`+` → `/surveys/new`(빌더)로 바로 진입한다.

- 문항 유형 7종: 단답형·장문형·객관식·체크박스·드롭다운·선형 배율·날짜.
- 질문별 편집(제목/유형/선택지 추가·삭제/필수 토글), 복제·삭제, 위·아래 이동 + 드래그 재정렬,
  실시간 미리보기, 발행 설정 시트(카테고리/목표/예상시간).
- 발행하면 피드에 노출되고, 응답자는 `/surveys/:id/fill`에서 **앱 안에서 바로 응답**한다.
  제출(필수 항목 검증)이 완료 근거이므로 증빙 토큰·외부 링크가 필요 없고, §4.1 정산(중복 차단·적립/차감)이
  그대로 적용된다. 무성의 방어용 최소 체류시간은 자체 빌더에선 5초 한도만 둔다(실제 답안이 핵심 신호).
- 시드의 모든 설문에 문항이 들어 있어 피드 어느 설문이든 바로 응답을 체험할 수 있다.

## UX 설계 (프로덕션 디테일)

- **접근성 있는 오버레이**: 공용 `Sheet`/`ConfirmDialog`가 포커스 트랩 · ESC 닫기 · 스크롤 잠금 ·
  닫힐 때 포커스 복원을 보장(`components/modal.ts`). 신고·발행 시트가 이 위에서 동작.
- **명령형 확인 다이얼로그**: `await confirmDialog({...})` 패턴(전역 `ConfirmHost`). 파괴적 작업에 적용 —
  질문 삭제, 로그아웃, 빌더 이탈.
- **작성 중 이탈 방지**: 빌더에서 변경분이 있으면 뒤로가기/탭 닫기 시 확인(`beforeunload` + 인앱 가드).
  발행 성공 시 경고 해제.
- **이동 시 포커스·스크롤**: 라우트 전환에 스크롤 상단 이동 + 주요 영역 포커스(스크린리더). 뒤로가기(POP)는
  사용자의 기존 위치를 존중해 강제하지 않음(`ScrollToTop`).
- **일관된 상태**: 피드·내 설문·설문 상세·크레딧에 로딩 스켈레톤 / 빈 상태 / 오류+재시도(`Skeleton`,
  `ErrorState`)를 통일.
- **사전 게이팅**: 잔액이 부족하면 설문 등록 진입에서 먼저 안내(빌드 노력 낭비 방지).
- 모든 모달/시트/스켈레톤은 `prefers-reduced-motion` 존중, 48px 터치 타깃, 키보드 포커스 가시.

## 디자인 (design_system.md + 시안)

- **그린 단색** 액션 컬러 + 잉크 네이비 중립, **화이트 기반** 배경. **semantic 토큰만** 참조
  (브랜드 primitive만 teal→green으로 교체해 토큰 아키텍처는 유지).
- 본문 Pretendard / 디스플레이 Wanted Sans / 모노 JetBrains Mono. 디스플레이는 화면당 한 곳.
- 시그니처 **원형 상호성 게이지**(반원 호 + 스마일 얼굴, 균형 시 펄스 1회)만 특별 모션 허용.
- 다크 테마(마이페이지 토글), `prefers-reduced-motion` 존중, 48px 터치 타깃, 키보드 포커스 가시.

## 백엔드 연동 시

`src/api/mockApi.ts`만 실제 HTTP 클라이언트(ky/axios, JWT 인터셉터)로 교체하면 된다.
함수 시그니처와 반환 타입은 그대로 유지하도록 설계했다.
